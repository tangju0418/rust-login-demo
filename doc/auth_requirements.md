# 认证模块业务规则

> 版本: v2.2
> 更新日期: 2026-03-24

---

## 一、数据结构

### 用户

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| 用户 ID | 整数 | 是 | 唯一，自动生成，不可修改 |
| 邮箱 | 字符串 | 是 | 唯一，最长 254 字符，合法邮箱格式 |
| 密码哈希 | 字符串 | 是 | 不可逆哈希，禁止明文或可逆加密 |
| 状态 | 枚举（`active` \| `disabled` \| `deleted`） | 是 | 默认 `active`；`disabled`：账号被暂停；`deleted`：软删除，记录保留 |
| 创建时间 | 时间戳 | 是 | — |
| 更新时间 | 时间戳 | 是 | 每次字段变更时更新 |

### Refresh Token 记录

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| 记录 ID | 整数 | 是 | 唯一，自动生成 |
| 所属用户 ID | 整数 | 是 | 关联用户 |
| 令牌哈希 | 字符串 | 是 | 唯一，服务端不持有原始值 |
| 过期时间 | 时间戳 | 是 | 签发时间 + Refresh Token 有效期 |
| 创建时间 | 时间戳 | 是 | — |
| 撤销时间 | 时间戳 | 否 | 为空：有效；有值：已撤销 |
| 来源 IP | 字符串 | 否 | 审计用 |
| 客户端标识 | 字符串 | 否 | 审计用 |

### 登录失败记录

| 字段 | 类型 | 必填 |
|---|---|---|
| 来源 IP | 字符串 | 是 |
| 尝试邮箱 | 字符串 | 是 |
| 发生时间 | 时间戳 | 是 |
| 失败原因 | 枚举（`user_not_found` \| `wrong_password`） | 是 |

### Access Token 载荷

| 字段 | 类型 | 必填 | 含义 |
|---|---|---|---|
| `sub` | 字符串 | 是 | 用户 ID |
| `email` | 字符串 | 是 | 用户邮箱 |
| `iat` | 整数 | 是 | 签发时间（Unix 时间戳） |
| `exp` | 整数 | 是 | 过期时间（Unix 时间戳） |
| `jti` | 字符串 | 是 | 令牌唯一标识 |
| `rtid` | 整数 | 是 | 绑定的 Refresh Token 记录 ID |

## 二、登录

### 输入

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| 邮箱 | 字符串 | 是 | 合法邮箱格式，最长 254 字符 |
| 密码 | 字符串 | 是 | 非空，最长 128 字符 |

### 判定（按优先级，任意一项不通过即终止）

| 优先级 | 条件 | 结果 |
|---|---|---|
| 1 | 该 IP 在最近 15 分钟内失败次数 ≥ IP 失败阈值 | 拒绝，附剩余封禁秒数 |
| 2 | 该邮箱在最近 30 分钟内失败次数 ≥ 邮箱失败阈值 | 拒绝，附剩余锁定秒数 |
| 3 | 邮箱不存在于用户表 | 写入失败记录（`user_not_found`），拒绝 |
| 4 | 用户状态 ≠ `active` | 写入失败记录（`user_not_found`），拒绝 |
| 5 | 密码与哈希不匹配 | 写入失败记录（`wrong_password`），拒绝 |
| 6 | 全部通过 | 签发 Access Token 和 Refresh Token |

### 约束

- 优先级 3、4、5 的失败须返回完全相同的错误文本，不得区分（防用户枚举）
- 优先级 3、4、5 的响应耗时须一致，即使用户不存在或已禁用也须执行等价密码比对（防时序攻击）
- 优先级 1、2 拦截时不执行密码比对，且不写入失败记录
- 封禁期间的新尝试照常写入失败记录，封禁时长不因新尝试延长
- 登录成功不主动清除失败记录，窗口滑动后旧记录自然失效

## 三、令牌刷新

### 输入

| 字段 | 类型 | 必填 |
|---|---|---|
| Refresh Token 原始值 | 字符串 | 是 |

### 有效性判定（全部满足才允许刷新）

- 令牌哈希在记录中存在
- 撤销时间为空
- 当前时间 < 过期时间
- 对应用户存在且状态为 `active`

### 重用检测

令牌哈希存在但撤销时间已有值（已被消费过），视为疑似泄露：撤销该用户全部 Refresh Token，拒绝本次请求。

### 成功输出

- 旧令牌立即撤销，无宽限期
- 签发新的 Access Token 和 Refresh Token
- 新 Refresh Token 有效期从新签发时间起算，不继承旧令牌剩余时间

---

## 四、登出

### 输入

- 有效的 Access Token（签名正确且未过期）

### 规则

- 从 `rtid` 定位对应 Refresh Token 记录，将其撤销时间设为当前时间
- 幂等：同一 Access Token 重复登出均成功

### 边界

- 仅撤销当前 `rtid` 绑定的 Refresh Token，不影响该用户其他设备
- Access Token 本身不可主动失效，依赖 2 小时有效期自然限制风险窗口

---

## 五、防暴力破解

### 判定维度

两个维度独立判定，任意一个满足即拒绝。统计窗口与封禁时长为同一值：在过去 N 秒内失败次数达到阈值，则封禁仍在持续，直至最早那条失败记录滑出窗口。

| 维度 | 统计窗口 / 封禁时长（默认） | 失败阈值（默认） |
|---|---|---|
| 客户端 IP | 15 分钟（900 秒） | 20 次 |
| 请求邮箱 | 30 分钟（1800 秒） | 10 次 |

**剩余封禁秒数** = 封禁时长 − (当前时间 − 窗口内最早失败记录的发生时间)

### 计数规则

- 仅登录操作的失败计入，刷新和登出不计入
- `user_not_found` 和 `wrong_password` 均计入，无区别

### 客户端 IP 来源（多源选择）

- 请求经过反向代理时，以代理传递的原始客户端 IP 为准
- 无代理时，以直连地址为准
- 封禁以单个 IP 为单位，不按网段聚合

---

## 六、令牌有效期

| 令牌 | 有效期（默认） | 起算点 |
|---|---|---|
| Access Token | 7200 秒 | 签发时间 |
| Refresh Token | 7 天 | 签发时间 |

- Access Token 过期后须通过 Refresh Token 换取，不可续期
- 每次刷新后，新 Refresh Token 有效期从新签发时间起算

---

## 七、Refresh Token 存储约束

- 服务端仅存哈希，原始值签发后不再持有
- 客户端应将原始值视为不透明字符串（黑盒）

---

## 八、数据保留

| 数据 | 可删除条件 |
|---|---|
| 登录失败记录 | 发生时间超出最长统计窗口（30 分钟）后 |
| Refresh Token（已过期） | 过期后可删除 |
| Refresh Token（已撤销） | 撤销后满 7 天可删除 |

---

## 九、安全约束

- 密码以不可逆哈希存储，禁止明文或可逆加密
- Refresh Token 服务端只存哈希，禁止存原始值
- JWT 签名密钥不得硬编码，须通过外部配置注入
- 日志中禁止出现明文密码或完整 Token 值

---

## 十、可配置参数

以下参数影响业务判定，须支持不修改代码的方式变更：

| 参数 | 默认值 |
|---|---|
| Access Token 有效期 | 7200 秒 |
| Refresh Token 有效期 | 7 天 |
| IP 失败阈值 | 20 次 |
| 邮箱失败阈值 | 10 次 |
| IP 统计/封禁窗口 | 900 秒 |
| 邮箱统计/封禁窗口 | 1800 秒 |

## 十一、测试环境约束

- 测试账号仅在账号数据为空时自动创建，生产环境须禁用

## 十二、验收 UI

验收 UI 为独立页面，不依赖外部工具，仅通过页面操作即可完成所有验收项的人工核查。

### 整体风格

- **PC 端布局**：最小宽度 1024px，居中容器，左右两栏或顶部标签页结构
- **清新风格**：白色/浅灰背景，蓝色主调，圆角卡片，无多余装饰
- 字体使用系统无衬线字体，正文 14px，标题 16–18px
- 各功能区以卡片形式分隔，卡片间距充足、层次清晰

### 必须支持的操作

| 操作 | 输入 | 成功展示 | 失败展示 |
|---|---|---|---|
| 登录 | 邮箱、密码 | 令牌摘要、有效期 | 错误信息、错误类型 |
| 刷新令牌 | Refresh Token（可手动修改） | 新令牌摘要、有效期 | 错误信息、错误类型 |
| 登出 | 无（使用当前 Access Token） | 成功提示 | 错误信息 |
| 解析令牌 | Access Token（可手动输入） | 载荷各字段、距过期剩余时间 | 解析失败提示 |
| 防暴力破解状态查询 | 邮箱 / IP（可分别输入） | 当前失败次数、距解封剩余时间 | — |

- 令牌摘要只展示部分内容，不展示完整值

### 防暴力破解专区

页面须包含独立的「防暴力破解」卡片，提供以下能力：

**实时状态面板**

| 展示项 | 说明 |
|---|---|
| 当前请求 IP | 页面加载时自动填入，可手动覆盖 |
| IP 近 15 分钟失败次数 / 阈值 | 格式：`已失败 N / 20 次` |
| IP 剩余封禁时间 | 封禁中显示倒计时（秒级实时刷新）；未封禁显示「未封禁」 |
| 邮箱近 30 分钟失败次数 / 阈值 | 格式：`已失败 N / 10 次` |
| 邮箱剩余锁定时间 | 锁定中显示倒计时（秒级实时刷新）；未锁定显示「未锁定」 |

- 状态面板在登录操作完成后自动刷新一次
- 支持手动点击「刷新状态」按钮触发更新

**失败记录列表**

- 展示最近 20 条登录失败记录，字段：发生时间、来源 IP、尝试邮箱、失败原因
- 记录按时间倒序排列，超出部分不分页（仅展示最新 20 条）
- 登录操作完成后自动追加新失败记录（无需整页刷新）

**快速触发工具**

- 提供「连续失败 N 次」快捷按钮（N 可输入，默认 5），一键批量提交失败登录以快速触发封禁，便于验收
- 批量提交期间显示进度（`已提交 K / N`），全部完成后刷新状态面板

### 状态流转规则

- 登录成功后，令牌自动填入刷新/解析操作的输入区，无需手动复制
- 刷新成功后，新令牌替换页面内当前令牌
- 页面状态不持久化，关闭/刷新页面后清除

### 429 展示规则

- 触发 429 时，在对应操作卡片顶部展示醒目横幅，包含：封禁/锁定原因（IP 或邮箱）、剩余秒数实时倒计时
- 倒计时归零后横幅自动消失，按钮恢复正常状态
- 倒计时期间操作按钮保持可用（用于观察响应变化）
- IP 封禁与邮箱锁定同时触发时，横幅分行分别展示

### 防重复提交规则

- 请求进行中禁止再次提交（按钮变灰 + loading 状态），请求完成后恢复
- 批量触发工具执行期间，登录卡片提交按钮同步禁用

### 须可直接构造的测试场景

| 场景 | 构造方式 |
|---|---|
| 使用已消费的旧 Refresh Token | 刷新后将旧值粘回输入框再次提交 |
| 邮箱格式非法 | 直接输入非邮箱格式字符串 |
| 触发邮箱锁定 | 连续提交错误密码，或使用快速触发工具批量提交 |
| 触发 IP 封禁 | 使用快速触发工具批量提交，数量超过 IP 阈值（20 次） |
| 观察封禁期间新失败记录照常写入 | 封禁中继续提交，检查失败记录列表有新增但封禁时长不延长 |
| 验证封禁时长不因新尝试延长 | 封禁倒计时期间多次提交，确认倒计时不重置 |
| 查看令牌载荷 | 解析区自动填入当前令牌后提交 |

## 十三、验收标准

### 功能

- [ ] 正确凭据登录成功，返回有效 Access Token 和 Refresh Token
- [ ] Access Token 有效期为 2 小时
- [ ] Refresh Token 换取新令牌对后，旧令牌立即失效
- [ ] 同一邮箱 30 分钟内失败 10 次后，后续请求被拒绝并返回剩余锁定时长
- [ ] 同一 IP 15 分钟内失败 20 次后，后续请求被拒绝并返回剩余封禁时长
- [ ] 封禁/锁定期间新的失败尝试照常写入失败记录，封禁时长不因新尝试延长
- [ ] IP 封禁与邮箱锁定触发时，响应中均包含剩余秒数字段
- [ ] 验收 UI 防暴力破解专区实时展示 IP/邮箱失败次数及倒计时，与后端数据一致
- [ ] 用户不存在与密码错误的响应文本完全一致
- [ ] 使用已撤销的 Refresh Token 触发该用户全部令牌撤销
- [ ] 登出后当前设备 Refresh Token 失效，其他设备不受影响
- [ ] Access Token 的 `rtid` 可正确定位对应 Refresh Token 记录

### 安全

- [ ] 数据库中密码以哈希存储，无明文
- [ ] 数据库中 Refresh Token 以哈希存储，无原始值
- [ ] JWT 签名密钥通过配置注入，无硬编码
- [ ] 日志中不出现明文密码或完整 Token 值
